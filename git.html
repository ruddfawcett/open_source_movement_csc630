git cheatsheet

Resources Assignments git cheatsheet
  • git init: Create a new git repository in the working directory. Existing files will not be automatically added to the repository (i.e. they will not be tracked). If a git repository already exists in the working directory, it is reinitialized. More specifically, it creates the .git directory in the current directory with objects, refs/heads, and refs/tags. The reference HEAD will point to the new repository.
  • git clone This command clones a remote repository onto your local machine. It will create remote references for each branch in the repository. It will automatically configure remote.origin.url (for pushing) and remote.origin.fetch (for fetching).
  • git status: Tells you the tree and index status (i.e. whether you're ready to commit or not)
  • git add <filename>: Adds the changed/created files to the index found in the workspace, to stage the content for its future commit.
  • git rm <filename>: Remove a file from git's index, and remove it from the working tree if it exists there.
  • git reset: Resets all of the changes since the last commit match back to the last commit. The primary use is git reset --hard, which resets everything. <filename>
  • git commit: Saves the latest staged changes into your local repository. You can add a message with your commit describing your changes. Optional flag includes the -b flag which allows you to add a string message with the command. Internally, it also logs a commit object which includes information about the commit.
  • git diff: Shows changes between commits, commit and working tree etc: git diff [options] [<commit>] [--] [<path>…​]
  • git log: Using git log shows the commit history of the user. The command will show the author of the commit, the date of the commit, and a commit hash that you can use a git checkout command on to view a detached head version of the repo.
  • git checkout: Change the workspace (i.e. the current files in the directory) to those located at the specified commit. In essence, moves "you" (a.k.a. HEAD) to the commit. If you add the -b flag, it will create a new branch with the specified name: git checkout -b <new_branch_name>.
  • git branch <branchname>

    This command will create a new branch in the current repository, with the name 'branchname'. Without the option branchname parameter, it will list all branches.

  • git remote: Remotes are repositories other than the one on your local disk which you can push your changes into or pull from. Use git remote to manage these repositories. With no arguments, git remote will show a list of the existing remotes. To perform operations on these remotes, use subcommands like add, remove, rename, etc...
  • git push: Updates the references (HEAD, master, etc.) on the remote using local references. It sends objects the are present locally but not remotely in order to bring the remote up to date with your local copy.
  • git merge <branchname>: Fetch branches and all other objects from one or more repositories. Note: fetch does not merge or overwrite local files.
  • git merge <branchname>: Adds the changes from the given commits into the current branch. Specifically, this command replays the changes made on the named commit (i.e. git pull topic) and records the result in a new commit. This command is used by git pull to incorporate changes from another repository.
  • git fetch: Downloads objects and refs from another repository. Fetch refs from one or more other repositories, along with the objects necessary to complete their histories. The names of refs that are fetched, together with the object names they point at, are written to .git/FETCH_HEAD.
  • git pull <remote_repo> <remote_branch>: Runs git fetch (with the given parameters) followed by git merge to merge retrieved branch heads into the current branch.
